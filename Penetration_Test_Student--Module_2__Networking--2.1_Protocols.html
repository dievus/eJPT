2.1 Protocols


Introduction



• Computers communicate with each other using protocols
◇ Protocols ensure that differernt computers, using different hardware and software, can communicate
• Information is carried by packets between networked devices
• Packets are streams of bits running as electric signals on a physical media
◇ Wire or wireless
• Every Packet has a Header and Payload
◇ Header has a protocol-specific structure, ensuring that the receiving host interprets the payload and can handle the overall communication
◇ Payload is the actual information.
• IP header is at least 160 bits (20 bytes), and includes information to interpret contents of the IP packet
◇ First FOUR bits identify IP version, either IPv4 or IPv6
◇ 32 bits starting at position 96 represent SOURCE ADDRESS
◇ Following four bytes represent the DESTINATION ADDRESS
◇ Using Hheader information, nodes involved can understand and use IP packets

Four Layers in TCP/IP model include:


• Application
• Transport
• Network
• Physical

Seven layer OSI model includes:


• Application
• Presentation
• Session
• Transport
• Network
• Data Link
• Physical

Encapsulation


• Entire upper protocol packet (header plus payload) is the payload of the lower layer; this is called ENCAPSULATION
• During encapsulation every protocol adds its own header to the packet, treating it as a payload
• This happens to every packet sent by a host
• Receiving host does this in reverse order
