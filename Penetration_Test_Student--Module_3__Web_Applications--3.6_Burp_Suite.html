3.6 Burp Suite


How does Burp Suite support my pentest career?



• Web app analysis
• Finding vulnerabilities
• Attacks
• Burp Suite is one of the most used penetration testing tools

Intercepting Proxies



• Any web application contains objects like scripts, images, style sheets/css, client and server-side intelligence
◇ Having tools that help study and analyze web application behavior is vital
• An intercepting proxy is a tool that lets you analyze and modify any request, and any request exchanged between an HTTP client and server
◇ By intercepting HTTP messages, pentesters can study web app behavior and test for vulnerabilities
• Most used web applications are:
◇ Intercepting proxy feature of Burp Suite
◇ ZAP
• Proxies are fundamental while analyzing web applications
• Do not confuse intercepting proxies with common web proxies, such as Squid
◇ Proxy servers have different purposes - bandwidth optimization, content filtering, ect.



Burp Suite Proxy



• Burp Suite allows you to:
◇ Intercept of requests and responses between web server and browser
◇ Build requests manually
◇ Crawl a website by automatically visiting every page found in a sit
◇ Fuzz web apps by sending patterns of valid and invalid inputs to test behavior
◇ By using Burp, you can intercept and modify requests coming from your browser before being sent to the remote server
▪ Headers and body of a message can be modified manually or automatically
• Find me in Proxy > Intercept
◇ Modify me in Proxy > Options

Burp Proxy Configuration



• You do not need to manually intercept and forward every request
◇ Even if you leave master interception off, Burp still collects information on the HTTP traffic coming to and from the browser
• Can check what Burp is collecting in two ways
◇ On the Proxy > History tab

◇ In the Target > Site Map tab


Burp Repeater



• Allows you to manually build raw HTTP requests
• Can do the same thing by using other tools such as netcat or telnet, but Burp provides you:
◇ Syntax highlighting
◇ Raw and rendered responses
◇ Integration with other Burp tools
• To create a request, first set the target by clicking the target/pencil icon in right corner
◇ Set/specify target host, port, and HTTPS
◇ Define request by using the Request text area
▪ Each request must have at least an HTTP VERB (GET, POST, HEAD, ETC)
▪ Host: header
▪ FOLLOWED BY TWO EMPTY LINES
◇ When request is complete, click Go to send it to the server
▪ Burp will display the response in the Response panel
• An easier method to build requests is to intercept a browser request with the proxy and send it to the repeater function
◇ CTRL+R shortcut, or right click the request body and select Send to Repeater
