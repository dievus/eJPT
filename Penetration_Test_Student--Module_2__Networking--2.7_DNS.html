2.7 DNS


Domain Naming System



• Domain Name System, or DNS, primarly converts human-readable names like www.elearnsecurity.com to IP addresses, and is a fundamental support protocol for the internet and computer networks in general.
• DNS name such as www.elearnsecurity.com or members.elearnsecurity.com can be broken down into the following parts:
◇ Top Level Domain (TLD)
◇ Domain
◇ Subdomain
◇ Host
• members.elearnsecurity.com
◇ members = host
◇ elearnsecurity = domain
◇ .com = TLD
• www.sub.domain.com
◇ www. = host
◇ sub = subdomain
◇ domain = domain
◇ .com = TLD
• Name resolution is performed by resolvers, which are servers that contact the top level domain DNS servers and follow the hierarchy of the DNS name to resolve the name of a host
• Resolvers are DNS servers provided by your ISP or publicly available like OpenDNS or Google DNS
• To convert a DNS name to an IP address, the operating system must contact a resolver server to perform the DNS resolution
• Resolver breaks down the DNS name in its parts and uses them to convert a DNS name to an IP address
• DNS Resolution Protocol
◇ Resolver contacts one of the root name servers; these contain info about top level domains
◇ It asks the TLD name server what's the name server that can give information (authoritative name server) about the domain the resolver is looking for
◇ If there are one or more subdomains, step 2 is performed again on the authoritative DNS server for every domain
◇ Finally, resolver asks for the name resolution of the host part
• DNS can also convert IP address to a DNS name via Reverse DNS Resolution
